# Change: Add PWA installability and push notifications

## Why

Users on mobile have no way to install the app to their home screen or receive notifications when a chat response completes. Long-running agent queries (multi-tool chains across DataGov + CBS) can take 30-60 seconds — if the user switches away from the browser tab, they have no way to know the answer has arrived. Making the app a PWA with push notifications solves both problems: installability gives a native-app feel, and push notifications re-engage users when answers are ready.

## What Changes

- Add a **web app manifest** (`app/manifest.ts`) with Hebrew RTL metadata, standalone display, and PWA icons
- Generate and serve **PWA icon assets** (192x192, 512x512) from the existing logo
- Create a **service worker** (`public/sw.js`) that handles push events and notification clicks (deep-links to `/chat/{threadId}`)
- Add a **`push_subscriptions` table** in Convex to persist Web Push subscriptions per user/guest
- Add **VAPID key configuration** for the Web Push API
- Install `web-push` server-side package for sending push notifications
- Add a **push notification trigger** in the `onFinish` callback of `app/api/chat/route.ts` — fires when a chat stream completes
- Create a **`usePushSubscription` hook** that registers the service worker, manages subscribe/unsubscribe, and stores the subscription in Convex
- Add a **notification permission prompt** UI component (Hebrew, RTL) — shown contextually, not on first visit
- Add **smart notification logic** in the service worker: skip notification if user has the chat tab focused, use `tag` for dedup
- Update `app/layout.tsx` with `theme-color` meta tag (manifest link is auto-generated by Next.js)
