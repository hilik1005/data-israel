## ADDED Requirements

### Requirement: Web App Manifest
The system SHALL serve a valid web app manifest at `/manifest.webmanifest` via the Next.js `app/manifest.ts` file convention. The manifest SHALL include the app name in Hebrew ("דאטה ישראל"), standalone display mode, RTL direction, Hebrew language, theme colors matching the app's dark/light themes, and references to PWA icon assets (192x192 and 512x512 PNG).

#### Scenario: Manifest served with correct metadata
- **WHEN** a browser requests `/manifest.webmanifest`
- **THEN** the response is a valid JSON manifest with `name: "דאטה ישראל"`, `display: "standalone"`, `dir: "rtl"`, `lang: "he"`, `start_url: "/"`, and icon entries for 192x192 and 512x512 sizes

#### Scenario: Manifest includes theme colors
- **WHEN** the manifest is parsed by the browser
- **THEN** `theme_color` and `background_color` SHALL be set to values consistent with the app's theme system

### Requirement: PWA Icon Assets
The system SHALL include PNG icon files at `public/icon-192x192.png` and `public/icon-512x512.png` suitable for home screen installation on mobile devices. Icons SHALL be derived from the existing application logo.

#### Scenario: Icons are accessible
- **WHEN** a browser requests `/icon-192x192.png` or `/icon-512x512.png`
- **THEN** the server responds with a valid PNG image of the correct dimensions

### Requirement: Service Worker Registration
The system SHALL register a service worker (`/sw.js`) from client-side code. Registration SHALL occur after the page has loaded and SHALL use `updateViaCache: 'none'` to ensure the latest service worker is always fetched.

#### Scenario: Service worker registers successfully
- **WHEN** the app loads in a browser that supports service workers
- **THEN** `navigator.serviceWorker.register('/sw.js')` is called and the service worker enters the `active` state

#### Scenario: Service worker registration skipped on unsupported browsers
- **WHEN** the browser does not support service workers (`'serviceWorker' not in navigator`)
- **THEN** no registration is attempted and no errors are thrown

### Requirement: Layout Metadata for PWA
The `app/layout.tsx` SHALL include a `theme-color` meta tag matching the app's theme. The manifest link is auto-generated by Next.js from `app/manifest.ts` and does not need to be manually added.

#### Scenario: Theme color meta tag present
- **WHEN** the HTML document is rendered
- **THEN** a `<meta name="theme-color">` tag is present with the app's primary theme color

### Requirement: Service Worker Security Headers
The `next.config.ts` SHALL define response headers for `/sw.js` including `Content-Type: application/javascript; charset=utf-8`, `Cache-Control: no-cache, no-store, must-revalidate`, and a strict `Content-Security-Policy`.

#### Scenario: Service worker served with security headers
- **WHEN** the browser requests `/sw.js`
- **THEN** the response includes `Cache-Control: no-cache, no-store, must-revalidate` and `Content-Type: application/javascript; charset=utf-8`
